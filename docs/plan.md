# Plan: Episode Mapping + PDF Source + Enrichment API (SHRUTI)

## Context
SHRUTI — Ramayan quiz app. Two parallel workstreams merged here:

1. **PDF extraction** — replace Wikisource crawler with local Griffith PDF (24869-pdf.pdf, 1960 pages). Pre-extract per-canto text files once, read on demand at generate time.
2. **Enrichment API** — 7 features that transform quiz tool into immersive learning: knowledge enrichment, scene context, bookmarks, daily ritual, narrative thread, journey progress, daily shloka.

UI (Flutter) and admin tool (Streamlit) are separate repos. This plan covers **content_generation repo only**: PDF extractor, DB migrations, FastAPI layer.

- Running headers: Left pages = `Canto VIII. Sumantra's Speech.` | Right = `The Ramayana` + page num
- Book headers: `Book I.` on first page of each book
- Boundary detection: regex `Canto\s+([IVXLCDM]+)\.` on first 300 chars per page; inherit previous on miss
- Book page starts: I=p24, II=p321, III=p827, IV=p1154, V=p1412, VI=p1524
- **489 canto groups extracted** from 1960 pages (minor gaps in Sundara 59-65 and some Yuddha — acceptable)

## Part 1 — PDF Extraction

### `scripts/extract_pdf.py` (run once)

```
python scripts/extract_pdf.py --pdf 24869-pdf.pdf --out canto_cache/
```

Algorithm (verified against actual PDF):
- Scan every page; detect book (`^Book\s+[IVXLCDM]+\.`) and canto (`Canto\s+[IVXLCDM]+\.`) in first 300 chars
- Inherit previous page's (book, canto) when no header found
- Group page texts by (book_int, canto_int)
- Write `canto_cache/Book_I/Canto_8.txt` etc. (Arabic numerals)
- Roman numeral converter built-in

## Part 4 — Service Layer

### `services/streak.py`

```
# STREAK LOGIC — GENTLE / PARENT-FRIENDLY
# - weekly_active_days = unique days active in last 7 calendar days
# - current_streak_days = consecutive days active up to today
# - GRACE DAY: 1 missed day does NOT break streak
#   (streak breaks only if 2+ consecutive days missed)
# - streak never silently decremented in background —
#   only recalculated on next user action (POST /user-progress/answer)
# - UI copy: "4 of 7 days this week" not "X day streak 🔥"
```

Functions:
- `calculate_streak(last_active_date, current_streak) -> int` — applies grace day rule
- `calculate_weekly_active(activity_dates: list[date]) -> int` — unique days in last 7

### `services/shloka_rotation.py`

```python
def get_todays_shloka_index(device_id: str, total: int) -> int:
    # hash(device_id + date.isoformat()) % total
    # Same user sees same shloka all day
    # Different users see different shlokas
    import hashlib, datetime
    key = device_id + datetime.date.today().isoformat()
    return int(hashlib.md5(key.encode()).hexdigest(), 16) % total
```

### `services/journey_calculator.py`

Functions:
- `get_current_position(user_progress, episodes) -> dict`
- `get_milestones_reached(episode_progress_map) -> list[str]` — maps completed episodes to story beat labels
- `get_next_milestone(current_episode_id, episodes) -> str`
- `get_week_summary(weekly_active_days) -> str` — returns human copy

## Part 8 — Tests

### `tests/test_enrichment.py`

Five test groups (pytest):
1. **Saved questions** — save, re-save (idempotent note update), list, unsave, list again
2. **Shloka rotation** — same device+date → same index; same device different date → different index; different devices same date → likely different
3. **Streak grace day** — 0 days missed → streak +1; 1 day missed → streak holds; 2 days missed → streak resets to 1
4. **Journey progress** — given mock episode_progress, correct current_position and milestones
5. **Home endpoint** — response has all required keys; works with no prior progress (new device)

Use SQLite in-memory DB for all DB tests. No network calls.
